# 路线图

本文记录 eMall 从基础交易系统到生产级大型电商平台的阶段规划。P0 到 P8 已作为主要工程阶段推进；
P9-lite、P10-lite、P14-lite 是稳定运行所需的轻量补强；P11 到 P13 属于更长期的大型平台能力。

## P0：生产导向基础

目标：让工程不是空骨架，而是能完成基本交易闭环。

已完成：

- Java 17 多模块 Maven 工程。
- 网关、用户、商品、库存、订单、购物车、支付、营销、搜索、履约、评价、售后等核心模块。
- 基础 REST API、统一响应、错误处理、ID 生成、异常处理。
- MySQL/Flyway 基线和本地内存模式。
- 幂等下单、库存预占、支付确认、取消释放库存。
- Java smoke 模块和基础单元测试。

## P1：高并发和可靠性加固

目标：让核心交易链路具备高并发和故障恢复思路。

已完成：

- 网关限流、服务容错、降级、重试、隔离设计。
- 库存热点 SKU 桶。
- 下游自适应平滑恢复。
- Outbox、补偿任务、支付回调幂等。
- 支付流水和渠道对账。
- Redis 缓存和缓存 TTL 基线。

## P2：生产运维基线

目标：让系统具备启动、部署、监控和验证基础。

已完成：

- Docker Compose 本地运行拓扑。
- Kubernetes Deployment、Service、PDB、HPA、探针、资源配置和优雅关闭。
- Prometheus、Grafana、日志、OpenTelemetry 基线。
- SLO、告警和故障处理手册。
- 压测工具和容量基线模板。
- CI、Checkstyle、Maven profile 和构建验证。

## P3：规模、多区域和商业成熟度

目标：补齐大规模电商中常见的业务和平台能力。

已完成：

- 商家、秒杀、推荐、实验、广告、成本治理。
- 多区域路由策略和单写归属模型。
- 混沌工程目录和安全门禁。
- 更完整的履约、售后、搜索和评价边界。

## P4：身份、信任、风控和平台治理

目标：建立内部和外部平台的信任边界。

已完成：

- `identity`：账号、会话、权限、服务客户端、商家子账号。
- `risk`：风控规则、设备信誉、风险事件和反作弊决策。
- `operations`：审批、运维任务、合规证据、安全事件。
- `openapi`：商家应用、签名校验、配额、Webhook。

## P5：商品、营销、搜索、推荐和实验平台

目标：让增长、运营和推荐能力独立演进。

已完成：

- `catalog`：类目、属性、品牌授权、SPU/SKU 治理。
- `promotion`：活动、叠加优惠、预算和日历。
- `experiment`：实验、互斥组、流量分配、护栏和报告。
- `advertising`：广告活动、素材、关键词、排序和扣费。

## P6：供应链、财务和客服深度

目标：补齐交易后的供应、财务和服务闭环。

已完成：

- `supply-chain`：入库、上架、调拨、运单、异常、签收凭证。
- `finance`：账户、流水、冻结、结算、发票、清分、拒付。
- `customer-service`：工单、分派、仲裁、补偿、知识库、服务评价。
- `forecasting`：需求信号、预测、补货、产能和风险汇总。

## P7：数据、AI、分析和决策平台

目标：让数据平台和智能决策具备独立模块边界。

已完成：

- `event-platform`：事件 schema、埋点采集、offset 和指标物化。
- `data-warehouse`：ODS、DWD、DWS、ADS、分区、质量检查、血缘。
- `intelligence`：画像、在线特征、模型部署和 AI 决策记录。
- `analytics`：指标、看板、异常、授权和隐私请求。

## P8：生产规模控制面

目标：补齐多活、可靠性、发布和平台运维控制能力。

已完成：

- `traffic`：单元、分片路由、流量切换、隔离。
- `reliability`：容量演练、SLO、混沌审批、上线门禁。
- `release`：功能开关、灰度、Topic 治理、回放计划。
- `platform-ops`：备份、数据库操作、FinOps、安全操作。

## P9-lite：稳定运行控制和护栏

目标：不做完整大厂控制台，但补齐稳定运行所需最小能力。

已完成：

- 内部运维接口安全基线。
- Outbox 重放和补偿入口。
- 运行时保护测试。
- 生产控制面基础模型。

## P10-lite：稳定数据一致性和任务协调

目标：补齐数据一致性和定时任务的最小生产护栏。

已完成：

- 幂等记录和数据库 Outbox。
- JDBC 分布式任务锁集成测试。
- 核心 Flyway schema 集成测试。
- Kafka Outbox 和 Redis 原语集成测试。

## P14-lite：稳定运行验证

目标：证明当前工程能够被持续验证。

已完成：

- 所有模块单元测试基线。
- 所有模块集成测试基线。
- 生产级集成测试 PIT-00 到 PIT-17。
- `mvn test` 和 `mvn verify -DskipITs=false` 可整体通过。

## P9：真实基础设施集成和操作控制台

后续可做：

- 真正的运维控制台 UI。
- 配置中心、服务发现和动态限流。
- 真实 Kubernetes 环境中的 HPA、PDB、滚动发布、自动回滚。
- 多环境发布流程和权限审批。

## P10：分库分表、高规模存储和数据一致性

后续可做：

- ShardingSphere 或自研分片路由。
- 订单、库存、支付等核心表分库分表。
- 历史数据归档和冷热分层。
- CDC、数据订正和一致性审计平台。

## P11：高级业务深度和商家运营

后续可做：

- 更完整的商家后台。
- 商品治理、平台招商、合同、保证金、处罚。
- 复杂售后、逆向物流、客服质检。
- 多租户计费和商家结算规则。

## P12：搜索、推荐、广告和 AI 生产化

后续可做：

- 更真实的召回、粗排、精排链路。
- 特征平台、模型训练、模型发布和回滚。
- 广告计费、预算控制和归因。
- A/B 实验平台的统计显著性和自动决策。

## P13：安全、合规、隐私和供应链安全

后续可做：

- 更完整的隐私合规、数据分级、脱敏和访问审计。
- SSO、MFA、细粒度权限和零信任接入。
- SBOM、依赖漏洞扫描、镜像签名和供应链安全。
- 税务、财务、支付牌照和地区合规适配。

## P14：生产验证、容量认证和发布就绪

后续可做：

- 长时间压测、容量模型和瓶颈报告。
- 故障演练、容灾演练和恢复时间验证。
- 真实环境端到端测试和数据对账。
- 上线前发布评审和回滚演练。
