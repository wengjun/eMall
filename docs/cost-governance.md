# 成本治理

成本治理用于在保证稳定性的前提下控制资源成本，避免系统为了“看起来生产级”而无限堆资源。

## 范围

当前 `cost` 模块关注：

- 服务预算。
- 缓存命中率。
- 索引增长。
- 存储冷热分层压力。
- 异步导出积压。
- 成本风险信号。
- 优化动作审批。

## 运行流程

1. 收集服务成本和资源使用信号。
2. 根据规则生成成本风险。
3. 去重生成优化动作。
4. 运维或 owner 审批动作。
5. 执行后记录效果和风险。

## 预算

预算不只是钱，还包括：

- CPU 和内存。
- 存储容量。
- 缓存容量。
- 搜索索引大小。
- MQ 积压。
- 离线任务执行时间。

预算超限不一定立即降配，需要结合业务峰值、SLO 和发布计划判断。

## 运维集成

成本治理应和以下能力结合：

- 监控告警。
- 容量压测。
- 发布审批。
- 数据归档。
- 缓存策略。
- 索引生命周期。

## 生产扩展

真实生产可以继续扩展：

- 云资源账单接入。
- 单服务成本分摊。
- 自动化扩缩容建议。
- 闲置资源检测。
- 大促前后容量回收。
- FinOps 报表和责任人机制。
