## 答案结构

- 先用通俗语言解释问题本质。
- 再说明生产系统里为什么重要。
- 然后给出 Java 或 eMall 项目的实现例子。
- 最后补充常见坑、面试追问和专家级表达。

## 学习方法

不要只背答案。专家级面试更关注你能不能把一个点讲到生产落地：

```text
问题本质 -> 业务影响 -> 实现方案 -> 失败场景 -> 监控恢复 -> 取舍演进
```

如果回答能自然关联到 eMall 的订单、库存、支付、Outbox、补偿、对账、限流、可观测性和测试，
通常会比纯理论答案更有说服力。

对高频题目，优先补充 Java 17 代码、SVG 图片、失败场景和监控恢复方式。
已增强题目见 [面试答案深度增强索引](deep-dive/README.md)。
跨题目复用的补强方法见 [面试补强共享框架](shared/deepening-framework.md)。
