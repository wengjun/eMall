# 297 事务 ACID 分别是什么？

[返回按分类学习面试题](../README.md)

## 题目

事务 ACID 分别是什么？

## 先给面试官的短答案

ACID 是事务的四个特性：原子性、一致性、隔离性、持久性。
原子性保证事务内操作要么都成功要么都失败；一致性保证数据从一个合法状态到另一个合法状态；
隔离性控制并发事务互相影响；持久性保证提交后的数据不会因崩溃丢失。

这些特性是核心交易数据可靠性的基础。

## Atomicity

原子性：

- 事务内操作整体成功或失败。
- 失败时回滚已执行操作。
- 通常依赖 undo log 实现。

例如订单创建和订单明细写入应一起成功或一起失败。

## Consistency

一致性：

- 数据满足约束和业务规则。
- 事务前后都处于合法状态。

例如库存不能扣成负数，支付金额不能和订单金额不一致。

## Isolation

隔离性：

- 并发事务之间互相隔离。
- 隔离级别不同，允许的并发现象不同。

它解决脏读、不可重复读和幻读等问题。

## Durability

持久性：

- 事务提交后数据可靠落盘。
- 即使数据库崩溃也能恢复。
- 通常依赖 redo log。

持久性是支付、订单等数据不可丢失的基础。

## 在 eMall 项目中怎么讲？

创建订单时，订单主表、订单明细和本地 Outbox 事件应在同一事务中提交。

如果任一步失败，整个事务回滚，避免只有订单没有事件或只有明细没有主表。

## 深度增强：Java 17 数据访问策略示例

```java
record QueryPlan(String accessPath, boolean usesIndex, boolean requiresPagination) {

    boolean safeForOnlineTraffic() {
        return usesIndex && requiresPagination;
    }
}

final class OnlineQueryPolicy {

    void verify(QueryPlan plan) {
        if (!plan.safeForOnlineTraffic()) {
            throw new IllegalArgumentException("Online query must use index and pagination");
        }
    }
}
```

这段代码体现线上查询治理：不是 SQL 能跑就可以上线，而是要确认走索引、可分页、可限流、可观测。

## 深度增强：生产边界

核心表设计要从典型查询倒推索引，避免全表扫描、深分页和大事务。分库分表要先选好分片键，
避免跨分片事务和热点分片。任何数据迁移都要支持灰度、校验、回滚或修复。

## 深度增强：面试高分表达

我会从访问模式回答数据题：谁查、按什么条件查、QPS 多少、数据量多大、是否强一致、是否需要分页和排序。
然后再决定索引、分片、缓存、读写分离和归档策略。

## 专家级完整回答

```text
ACID 是原子性、一致性、隔离性和持久性。原子性保证事务内操作整体提交或回滚；
一致性保证业务约束不被破坏；隔离性控制并发事务的相互影响；持久性保证提交结果崩溃后仍可恢复。

在电商核心交易中，订单、支付、库存的本地事务必须依赖 ACID 保证局部正确性。
```
