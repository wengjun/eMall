# 301 间隙锁是什么？

[返回按分类学习面试题](../README.md)

## 题目

间隙锁是什么？

## 先给面试官的短答案

间隙锁是 InnoDB 锁住索引记录之间范围的锁，用来防止其他事务在这个范围插入新记录，从而避免幻读。
它通常出现在可重复读隔离级别下的范围查询和当前读中，并且依赖索引范围。

间隙锁保护一致性，但也可能降低并发，甚至引发死锁。

## 锁住什么？

如果索引中有值 `10` 和 `20`，间隙可以理解为：

```text
(10, 20)
```

锁住这个间隙后，其他事务不能在这个范围插入新的索引值。

## 什么时候出现？

常见场景：

- `SELECT ... FOR UPDATE` 范围查询。
- `UPDATE` 范围条件。
- `DELETE` 范围条件。
- 可重复读隔离级别下防幻读。

是否加间隙锁和索引、隔离级别、查询条件有关。

## 风险

风险包括：

- 插入被阻塞。
- 锁范围过大。
- 并发下降。
- 死锁概率上升。

没有合适索引时，锁范围可能比预期大很多。

## 在 eMall 项目中怎么讲？

后台批量更新某时间范围订单状态时，如果使用范围当前读，可能锁住一段索引间隙，阻塞新订单插入或状态更新。

应避免在主交易表上执行大范围加锁操作。

## 深度增强：Java 17 数据访问策略示例

```java
record QueryPlan(String accessPath, boolean usesIndex, boolean requiresPagination) {

    boolean safeForOnlineTraffic() {
        return usesIndex && requiresPagination;
    }
}

final class OnlineQueryPolicy {

    void verify(QueryPlan plan) {
        if (!plan.safeForOnlineTraffic()) {
            throw new IllegalArgumentException("Online query must use index and pagination");
        }
    }
}
```

这段代码体现线上查询治理：不是 SQL 能跑就可以上线，而是要确认走索引、可分页、可限流、可观测。

## 深度增强：生产边界

核心表设计要从典型查询倒推索引，避免全表扫描、深分页和大事务。分库分表要先选好分片键，
避免跨分片事务和热点分片。任何数据迁移都要支持灰度、校验、回滚或修复。

## 专家级完整回答

```text
间隙锁是 InnoDB 锁住索引记录之间范围的锁，用于阻止其他事务插入满足范围条件的新记录，从而避免幻读。
它常出现在可重复读隔离级别下的范围当前读中，例如 select for update、update 和 delete。

间隙锁能保证一致性，但会降低并发。索引设计不合理或范围过大时，可能锁住大量数据并引发死锁。
```

## 回答评分点

高分答案应该覆盖：

- 间隙锁锁的是索引范围。
- 用于防止插入幻影记录。
- 常见于范围当前读。
- 依赖索引和隔离级别。
- 可能降低并发和引发死锁。
