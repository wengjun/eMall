# 284 聚簇索引和二级索引有什么区别？

[返回按分类学习面试题](../README.md)

## 题目

聚簇索引和二级索引有什么区别？

## 先给面试官的短答案

聚簇索引决定数据行的存储组织，InnoDB 的主键索引就是聚簇索引，叶子节点存整行数据。
二级索引是非主键索引，叶子节点存索引列和主键值，查询非索引字段时需要通过主键回表。

一个表只能有一个聚簇索引，但可以有多个二级索引。

## 聚簇索引

特点：

- 通常是主键索引。
- 叶子节点存整行。
- 数据按主键组织。
- 一个表只有一个。

主键查询通常非常高效。

## 二级索引

特点：

- 按非主键列建立。
- 叶子节点存索引列和主键。
- 查询整行可能需要回表。
- 可以有多个。

二级索引越多，写入维护成本越高。

## 查询差异

主键查询：

```sql
SELECT * FROM orders WHERE id = ?;
```

通过聚簇索引直接拿到整行。

二级索引查询：

```sql
SELECT * FROM orders WHERE user_id = ?;
```

先通过 `user_id` 索引找到主键，再用主键回表。

## 在 eMall 项目中怎么讲？

订单详情按订单 ID 查询走聚簇索引。

用户订单列表按 `user_id` 查询走二级索引，如果列表只返回索引覆盖字段，可以减少回表。

## 深度增强：Java 17 数据访问策略示例

```java
record QueryPlan(String accessPath, boolean usesIndex, boolean requiresPagination) {

    boolean safeForOnlineTraffic() {
        return usesIndex && requiresPagination;
    }
}

final class OnlineQueryPolicy {

    void verify(QueryPlan plan) {
        if (!plan.safeForOnlineTraffic()) {
            throw new IllegalArgumentException("Online query must use index and pagination");
        }
    }
}
```

这段代码体现线上查询治理：不是 SQL 能跑就可以上线，而是要确认走索引、可分页、可限流、可观测。

## 深度增强：生产边界

核心表设计要从典型查询倒推索引，避免全表扫描、深分页和大事务。分库分表要先选好分片键，
避免跨分片事务和热点分片。任何数据迁移都要支持灰度、校验、回滚或修复。

## 专家级完整回答

```text
聚簇索引是数据行的组织方式，InnoDB 中主键索引叶子节点存整行数据。
二级索引叶子节点存索引列和主键值，如果查询需要其他列，就要通过主键回表。

因此主键设计影响整张表和所有二级索引，二级索引设计要考虑回表成本和写入维护成本。
```

## 回答评分点

高分答案应该覆盖：

- 聚簇索引叶子节点存整行。
- 二级索引叶子节点存主键。
- 二级索引查询可能回表。
- 一个表只有一个聚簇索引。
- 二级索引会增加写入成本。
