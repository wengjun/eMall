# 099 多个下游服务是否应该共享同一个线程池？

[返回按分类学习面试题](../README.md)

## 题目

多个下游服务是否应该共享同一个线程池？

## 先给面试官的短答案

不应该无脑共享。低风险、低流量、相同 SLA 的下游可以共享；核心链路、慢风险高、容量差异大或故障影响大的下游应该隔离。
共享线程池会让一个慢下游占满线程，拖垮其他正常下游。

原则是按业务重要性、下游容量、延迟特征和故障隔离需求决定。

## 共享的好处

共享线程池有一些好处：

- 配置简单。
- 线程数较少。
- 资源利用率可能更高。
- 运维成本低。

对低价值、低风险、低流量任务可以接受。

## 共享的风险

主要风险是故障扩散。

例如库存服务变慢，占满共享线程池。

结果：

- 支付查询也无法执行。
- 用户查询也变慢。
- 售后查询也排队。
- 整个订单服务 P99 升高。

一个下游慢变成整个服务慢。

## 什么时候应该隔离？

应该隔离的场景：

- 核心交易链路。
- 下游容量差异大。
- 下游延迟波动大。
- 下游历史故障多。
- 调用量很高。
- 任务耗时差异大。
- 业务优先级不同。

例如支付、库存、风控通常不应该和推荐、广告共享同一个池。

## 隔离的代价

隔离不是越多越好。

代价包括：

- 线程总数增加。
- 配置复杂。
- 监控项增加。
- 资源利用率下降。
- 参数维护成本增加。

所以要按风险分级，不是每个接口一个线程池。

## 折中方案

可以按分组隔离：

- 核心交易池。
- 支付下游池。
- 库存下游池。
- 查询类共享池。
- 非核心推荐广告池。
- 后台任务池。

这样既控制复杂度，又能隔离核心风险。

## 在 eMall 项目中怎么讲？

订单创建中库存、支付、风控都影响交易成功率，应独立隔离。

订单详情中的物流、售后、推荐可以按优先级分组。

推荐服务慢时，可以降级为空，不应该影响订单基础信息展示。

## 深度增强：并发治理图

![Java 并发从线程安全到容量保护](../assets/concurrency-governance.svg)

并发题不能只回答 API 用法。生产系统要同时考虑线程安全、资源隔离、超时、拒绝、幂等和分布式多实例。
单机锁只能保护当前 JVM，不能保护整个集群；线程池满也不是小问题，而是容量和可用性风险。

## 深度增强：Java 17 有界并发示例

```java
import java.util.concurrent.Semaphore;
import java.util.function.Supplier;

final class BulkheadGuard {
    private final Semaphore permits;

    BulkheadGuard(int maxConcurrentCalls) {
        this.permits = new Semaphore(maxConcurrentCalls);
    }

    <T> T execute(Supplier<T> supplier) {
        if (!permits.tryAcquire()) {
            throw new IllegalStateException("Bulkhead rejected the call");
        }
        try {
            return supplier.get();
        } finally {
            permits.release();
        }
    }
}
```

这段代码展示了并发控制的生产思路：不是让所有请求无限进入系统，而是在入口保护共享资源。
真实项目还要加超时、指标、降级和按下游隔离。

## 深度增强：生产边界

线程安全不等于系统安全。`ConcurrentHashMap` 只能保护当前进程内的数据结构，
不能替代数据库唯一键、幂等表或分布式一致性。线程池也不能使用无界队列，
否则会把过载转化成内存上涨和 P99 恶化。

## 深度增强：面试高分表达

我会把并发问题分成三层：JMM 和锁保证单机正确性，线程池和隔离保证资源不被拖垮，
幂等和唯一键保证分布式正确性。这样能体现我理解 Java 并发，也理解微服务生产稳定性。

## 专家级完整回答

```text
多个下游不应该无脑共享线程池。共享能提高资源利用率和简化配置，但会导致慢下游占满线程，
影响其他正常下游。核心交易、容量差异大、延迟波动大、故障风险高的下游应独立隔离。

我会按业务优先级和风险分组隔离，而不是每个下游一个池。隔离要配合超时、限流、熔断和监控。
```

## 回答评分点

高分答案应该覆盖：

- 不能无脑共享。
- 共享会导致故障扩散。
- 核心和高风险下游要隔离。
- 隔离也有成本。
- 按风险和优先级分组。
