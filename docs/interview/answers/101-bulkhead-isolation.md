# 101 什么是舱壁隔离？

[返回按分类学习面试题](../README.md)

## 题目

什么是舱壁隔离？

## 先给面试官的短答案

舱壁隔离来自船舱设计：一个舱进水不能让整艘船沉没。软件系统中，它指把资源按业务、下游或优先级隔离，
让一个局部故障不会耗尽全部资源。常见实现包括线程池隔离、连接池隔离、信号量隔离、实例隔离和队列隔离。

它的目标是限制故障半径，保住核心能力。

## 为什么需要舱壁隔离？

大型电商系统依赖很多下游。

如果所有请求共享同一批资源：

- 库存慢会拖垮订单。
- 推荐慢会影响商品详情。
- 报表任务会挤占交易资源。
- 非核心任务会影响核心支付。

舱壁隔离就是防止这些问题互相传染。

## 常见隔离方式

常见方式：

- 线程池隔离。
- 信号量隔离。
- 数据库连接池隔离。
- HTTP 连接池隔离。
- MQ topic 或 consumer group 隔离。
- Kubernetes 资源隔离。
- 服务实例隔离。
- 队列隔离。

隔离可以发生在代码层、资源层和部署层。

## 线程池隔离

不同下游使用不同线程池。

库存线程池打满，不影响支付线程池。

适合阻塞调用和慢下游风险较高场景。

代价是线程数和配置复杂度增加。

## 信号量隔离

信号量隔离限制并发数，但不切换线程池。

适合：

- 短耗时任务。
- 非阻塞调用。
- 线程切换成本敏感场景。

它开销低，但隔离强度不如线程池隔离。

## 实例和资源隔离

核心交易服务可以和后台任务分开部署。

例如：

- 订单创建实例。
- 订单查询实例。
- 订单后台补偿实例。

不同实例设置不同 CPU、内存和扩缩容策略。

这比只在进程内隔离更彻底。

## 在 eMall 项目中怎么讲？

eMall 中推荐、广告、营销是非核心能力，订单、库存、支付是核心能力。

推荐服务慢时，商品详情可以降级不展示推荐，但不能影响商品基础信息和下单。

订单服务内部也要把库存、支付、风控调用隔离，防止一个下游拖垮全链路。

## 深度增强：并发治理图

![Java 并发从线程安全到容量保护](../assets/concurrency-governance.svg)

并发题不能只回答 API 用法。生产系统要同时考虑线程安全、资源隔离、超时、拒绝、幂等和分布式多实例。
单机锁只能保护当前 JVM，不能保护整个集群；线程池满也不是小问题，而是容量和可用性风险。

## 深度增强：Java 17 有界并发示例

```java
import java.util.concurrent.Semaphore;
import java.util.function.Supplier;

final class BulkheadGuard {
    private final Semaphore permits;

    BulkheadGuard(int maxConcurrentCalls) {
        this.permits = new Semaphore(maxConcurrentCalls);
    }

    <T> T execute(Supplier<T> supplier) {
        if (!permits.tryAcquire()) {
            throw new IllegalStateException("Bulkhead rejected the call");
        }
        try {
            return supplier.get();
        } finally {
            permits.release();
        }
    }
}
```

这段代码展示了并发控制的生产思路：不是让所有请求无限进入系统，而是在入口保护共享资源。
真实项目还要加超时、指标、降级和按下游隔离。

## 深度增强：生产边界

线程安全不等于系统安全。`ConcurrentHashMap` 只能保护当前进程内的数据结构，
不能替代数据库唯一键、幂等表或分布式一致性。线程池也不能使用无界队列，
否则会把过载转化成内存上涨和 P99 恶化。

## 深度增强：面试高分表达

我会把并发问题分成三层：JMM 和锁保证单机正确性，线程池和隔离保证资源不被拖垮，
幂等和唯一键保证分布式正确性。这样能体现我理解 Java 并发，也理解微服务生产稳定性。

## 专家级完整回答

```text
舱壁隔离是把资源按业务、下游或优先级拆开，限制故障半径。实现方式包括线程池隔离、信号量隔离、
连接池隔离、队列隔离、实例隔离和 Kubernetes 资源隔离。它不是为了提高单点性能，而是为了在局部
故障时保住核心能力。

在电商系统里，推荐广告失败可以降级，订单支付库存必须优先保护，所以资源隔离和降级策略要按业务优先级设计。
```

## 回答评分点

高分答案应该覆盖：

- 舱壁隔离是限制故障半径。
- 线程池、信号量、连接池、实例都可以隔离。
- 核心和非核心资源要分开。
- 隔离有成本，要按风险设计。
