# 523 设计价格服务

[返回按分类学习面试题](../README.md)

## 题目

设计价格服务。

## 先给面试官的短答案

价格服务负责商品基础价、活动价、会员价、促销优惠、运费、税费和最终应付金额的统一计算。核心要求是准确、
可解释、可审计、低延迟和可回放。下单时必须保存价格快照，后续支付、退款和对账都基于快照而不是实时价格。

## 核心职责

价格服务要提供商品展示价、购物车试算价、订单确认价和最终订单价。不同场景对实时性和准确性要求不同。

展示价可以短缓存，订单确认价必须强校验，最终订单价必须保存快照。

## 价格组成

价格通常由基础价、商家价、活动价、会员价、优惠券、满减、积分、运费、税费和平台补贴组成。

每一项都要有来源、规则 ID、计算过程和金额明细。最终金额不能是黑盒结果。

## 规则和一致性

价格规则要有版本号和生效时间。下单时使用的规则版本必须记录到订单快照中。

价格变更不能影响已创建订单，除非业务明确要求重新计价。支付和退款应该基于订单金额快照。

## 性能设计

价格查询频繁，可以对商品基础价和活动规则做缓存。复杂促销计算可以预编译规则或拆分为规则引擎。

但核心下单链路不能使用过期价格直接成交。缓存只能加速，不能替代最终校验。

## 在 eMall 项目中怎么讲？

eMall 的 `pricing` 模块负责金额汇总，`promotion` 负责促销规则，`order` 保存价格快照。
`payment` 和 `after-sales` 使用订单快照，避免实时价格变化影响资金链路。

## 专家级完整回答

```text
价格服务的关键不是把金额算出来，而是把金额算得准、解释得清、回放得出。

我会把价格拆成基础价、活动价、会员价、优惠、运费和税费等明细，每个明细都有规则版本和来源。
下单成功后保存价格快照，支付、退款、结算和对账都基于快照。

展示页可以缓存价格，但下单确认必须重新计算和校验。否则价格变更、促销失效或缓存旧值会直接造成资金风险。
```

## 回答评分点

高分答案应该覆盖：

- 覆盖展示价、试算价、确认价和最终订单价。
- 能拆解基础价、促销、运费、税费等组成。
- 强调规则版本、价格快照和可解释性。
- 知道支付和退款应基于订单快照。
- 能说明缓存加速但不能替代最终校验。

## 深度完善：系统设计高分展开

围绕「设计价格服务」，系统设计题要按目标、规模、架构、数据、失败路径、演进六层展开。
不要只画组件图，必须讲清每个组件为什么存在，以及它如何在大促和故障场景下保持可控。

### 容量和约束

- 先给假设：10 亿注册用户、1 亿 DAU、100 万峰值并发、核心链路优先保证可用性。
- 区分读多写少、写强一致、异步最终一致、离线分析和运维控制面。
- 明确 SLO，例如核心接口成功率、P99、恢复时间、数据可恢复窗口和成本上限。

### 白板架构

- 入口层：Gateway 做认证、签名、限流、灰度、请求体大小控制和 Trace 注入。
- 应用层：按领域拆服务，核心写链路尽量短，非核心能力通过事件异步解耦。
- 数据层：主库负责交易事实，缓存和搜索承担读流量，Outbox 保障事件可靠投递。
- 治理层：配置、发布、熔断、降级、告警、Runbook 和补偿平台形成闭环。

### 数据模型和数据流

核心表要能回答三个问题：当前状态是什么、为什么变成这样、失败后如何恢复。
通常至少需要业务主表、状态流水表、幂等表、Outbox 表、补偿任务表和审计表。

```sql
-- 示例：生产系统中不要只存当前状态，还要保留可追溯的事实和恢复入口。
create table operation_journal (
    id bigint primary key,
    business_key varchar(64) not null,
    operation_type varchar(64) not null,
    before_state varchar(64),
    after_state varchar(64),
    created_at timestamp not null
);
```

### 失败路径

- 下游慢：先超时、隔离和熔断，再用降级结果保护主链路。
- 消息失败：用 Outbox、重试、DLQ 和重放平台恢复。
- 数据不一致：用状态机、对账和补偿任务修复。
- 发布异常：用金丝雀、自动暂停、配置回滚和数据兼容策略控制影响面。

### 验收指标

- 用户侧：成功率、P95/P99、错误码分布、投诉量。
- 系统侧：QPS、CPU、连接池、队列积压、缓存命中率、MQ lag。
- 数据侧：对账差异、补偿积压、重复请求、非法状态迁移。
- 成本侧：单请求成本、存储增长、日志和 Trace 采样成本。
