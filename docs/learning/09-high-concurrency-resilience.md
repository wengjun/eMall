## 高并发和稳定性

### 限流

限流用于保护系统不被突发流量打垮。

常见维度：

- IP。
- 用户。
- 设备。
- SKU。
- 接口。
- 商家。

常见算法：

- 固定窗口。
- 滑动窗口。
- 令牌桶。
- 漏桶。

本工程网关可按组合 key 限流，例如：

```text
userId + deviceId + skuId + clientIp
```

### 熔断

熔断用于下游故障时快速失败。

状态：

- CLOSED：正常调用。
- OPEN：直接拒绝。
- HALF_OPEN：放少量探测流量。

关键点：

- 熔断保护调用方线程池。
- 半开状态不能一下子恢复全部流量。
- 恢复阶段失败要重新打开熔断。

### 降级

降级用于非核心能力失败时保护核心链路。

示例：

- 营销失败，下单按无优惠继续。
- 推荐失败，返回默认商品。
- 评价失败，商品详情不显示评分。
- 广告失败，不影响搜索自然结果。

不能随便降级的能力：

- 价格。
- 库存。
- 支付。
- 账户安全。

### 超时和重试

所有远程调用都必须设置超时。

重试原则：

- 只重试可幂等操作。
- 不要无限重试。
- 使用退避策略。
- 不要在高峰期放大故障。
- 写操作重试必须有幂等键。

### 热点 SKU

热点 SKU 问题：

- 所有人同时抢一个库存行。
- 数据库行锁竞争严重。
- 容易超时。

库存桶方案：

- 一个 SKU 拆成多个库存桶。
- 请求分散到不同桶。
- 每个桶独立扣减。
- 查询时汇总库存。

缺点：

- 实现复杂。
- 需要处理桶之间不均衡。
- 需要聚合查询。

### 秒杀削峰

秒杀常见方案：

- 活动预热。
- 库存预分配。
- 令牌发放。
- 用户限购。
- 请求排队。
- 异步创建订单。
- 独立秒杀链路。

核心目标：

- 不让秒杀流量打垮普通下单。
- 不让数据库直接承受全部请求。
- 提前拒绝没有资格的请求。
