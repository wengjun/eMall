## 推荐学习路径

### 第一阶段：能写稳定的单体服务

学习：

- Java 17。
- Spring Boot。
- REST API。
- MySQL。
- Flyway。
- Maven。
- JUnit。

练习：

- 写用户服务。
- 写商品服务。
- 写库存查询和初始化。
- 写统一响应和异常处理。

达标：

- 能独立写一个可测试、可启动、可维护的 Spring Boot 服务。

### 第二阶段：完成核心交易闭环

学习：

- 订单状态机。
- 库存预占。
- 支付单。
- 支付回调。
- 价格快照。
- 优惠快照。

练习：

- 写 `POST /api/orders`。
- 写库存预占、确认、释放。
- 写支付回调。
- 写订单取消。

达标：

- 能完整跑通下单、锁库存、支付、取消、释放库存。

### 第三阶段：补齐分布式可靠性

学习：

- 幂等。
- Outbox。
- Kafka。
- 补偿任务。
- 对账。
- 内部运维接口。

练习：

- 重复下单只生成一个订单。
- 重复支付回调只入账一次。
- Outbox 失败后可以重放。
- 库存失败后可以补偿。

达标：

- 能处理“中间失败后怎么恢复”。

### 第四阶段：补齐高并发和稳定性

学习：

- Redis 缓存。
- 网关限流。
- 熔断。
- 降级。
- 超时。
- 重试。
- 热点 SKU。
- 秒杀削峰。

练习：

- 给商品查询加缓存。
- 给网关加限流。
- 给营销失败加降级。
- 给库存加库存桶。

达标：

- 能解释如何保护核心交易链路。

### 第五阶段：补齐生产工程能力

学习：

- Docker。
- Docker Compose。
- Kubernetes。
- Prometheus。
- Grafana。
- OpenTelemetry。
- Testcontainers。
- Smoke 测试。
- 压测。
- Runbook。

练习：

- 本地启动完整依赖。
- 写集成测试。
- 写 smoke 测试。
- 写压测场景。
- 查看指标和日志。

达标：

- 系统能部署、能观测、能测试、能恢复。


## 面试表达重点

面试时可以这样讲：

```text
这个项目不是简单 CRUD，而是围绕大型电商的核心交易链路做工程化设计。
我重点实现和学习了订单、库存、支付之间的一致性问题，包括幂等、库存预占、
Outbox、补偿、支付对账、限流、熔断、降级、可观测和测试验证。
我不会说它已经真实承载京东级流量，但它能体现我对高并发、分布式一致性、
故障恢复和生产可运维性的理解。
```

高价值追问准备：

- 下单为什么要 `requestId`？
- 库存预占失败后订单怎么办？
- 支付成功但订单确认失败怎么办？
- MQ 消息重复消费怎么办？
- Outbox 为什么比直接发 MQ 更可靠？
- 营销服务挂了能不能继续下单？
- 价格服务挂了为什么不能继续下单？
- 热点 SKU 为什么会成为瓶颈？
- 怎么证明这些能力不是只写在文档里？
