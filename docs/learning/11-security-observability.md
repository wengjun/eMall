## 安全

### 敏感数据保护

敏感数据包括：

- 手机号。
- 身份证。
- 地址。
- 支付信息。
- token。
- 密钥。

保护方式：

- AES-GCM 加密。
- HMAC hash 精确查询。
- 日志脱敏。
- 最小权限。
- 审计。

手机号查询示例思路：

```text
mobile_plaintext -> HMAC hash -> 用 hash 查重或精确查询
mobile_plaintext -> AES-GCM -> 加密存储
```

### HMAC 签名

开放平台接口需要防篡改和防重放。

常见字段：

- appKey。
- timestamp。
- nonce。
- requestBody。
- signature。

校验逻辑：

1. 检查 appKey 是否存在。
2. 检查 timestamp 是否过期。
3. 检查 nonce 是否重复。
4. 用密钥重新计算签名。
5. 比较签名是否一致。

### RBAC

RBAC 包括：

- 用户。
- 角色。
- 权限。
- 资源。

示例：

- 运营可以审核商品。
- 财务可以处理结算。
- 客服可以处理工单。
- 运维可以触发补偿。

### 审计

需要审计：

- 内部补偿。
- Outbox 重放。
- 支付对账。
- 权限变更。
- 风控规则变更。
- 数据修复。

审计字段：

- 操作人。
- 操作时间。
- trace ID。
- 目标资源。
- 操作类型。
- 请求摘要。
- 结果。
- 失败原因。


## 可观测性

### 日志

日志要回答：

- 谁发起的请求？
- 请求处理到哪一步？
- 哪个订单或支付单失败？
- 失败原因是什么？
- trace ID 是什么？

日志原则：

- 结构化。
- 有 trace ID。
- 有业务 ID。
- 不输出敏感信息。
- 错误日志包含足够上下文。

### 指标

业务指标：

- 下单成功率。
- 支付成功率。
- 库存预占失败率。
- 退款失败率。
- 对账差异数量。

技术指标：

- QPS。
- P95/P99。
- 错误率。
- JVM 内存。
- GC。
- 线程池队列。
- 数据库连接。
- Kafka 积压。
- Redis 命中率。

### Trace

Trace 用于串起一次请求经过的所有服务。

必须做到：

- 网关生成或接收 `X-Trace-Id`。
- 内部服务继续透传。
- 日志写入同一个 trace ID。
- 下游失败能按 trace ID 查询。

### SLO 和 Runbook

SLO 是目标，例如：

- 下单可用性 99.9%。
- 支付回调可用性 99.99%。
- 下单 P99 小于 1000 ms。

Runbook 是故障处理手册，例如：

- 下单成功率下降先看什么。
- 库存失败率上升怎么处理。
- 支付回调积压怎么恢复。
- Outbox 失败如何重放。
